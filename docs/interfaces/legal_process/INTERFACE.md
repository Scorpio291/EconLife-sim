# Module: legal_process

## Purpose
Manages court proceedings, sentencing, appeals, and the full legal case lifecycle. Converts completed investigations into legal outcomes through a multi-stage state machine (investigation -> arrested -> charged -> trial -> convicted/acquitted -> imprisoned/fined -> paroled/pardoned). Evidence quality directly affects conviction probability. Player agency exists at each transition (bail, defense counsel, appeal). Processing runs at tick step 14. NOT province-parallel because the judiciary operates at the national level and legal cases can involve actors across multiple provinces.

## Inputs (from WorldState)
- `legal_cases[]` — all active LegalCase records: id, defendant_id, stage (LegalCaseStage), opened_tick, stage_entered_tick, evidence_weight, evidence_token_ids, lead_investigator_id, prosecutor_npc_id, judge_npc_id, charge_severity, sentence_ticks, sentence_remaining, bail_posted, appeal_pending, defence_quality
- `evidence_pool[]` — EvidenceToken records referenced by legal_cases[].evidence_token_ids
- `npcs[]` — NPC records for prosecutor, judge, investigator NPCs
- `player` — PlayerCharacter state for defendant-is-player cases
- `investigator_meters[]` — InvestigatorMeter records (status checked for arrest trigger)
- `config.legal.arrest_evidence_threshold` — 0.35; evidence weight for arrest
- `config.legal.charge_evidence_threshold` — 0.55; required to file formal charges
- `config.legal.dismissal_evidence_threshold` — 0.25; below this, case dismissed
- `config.legal.charge_to_trial_ticks` — 180; ~6 months from charges to trial
- `config.legal.investigation_to_charge_ticks` — 60; ~2 months from arrest to charges
- `config.legal.defence_evidence_discount` — 0.40; max discount from perfect defence
- `config.legal.base_sentence_ticks_per_severity` — 365; severity 1 = 1yr; severity 5 = 5yr
- `config.legal.custodial_sentence_severity_floor` — 3; below this, fine only
- `config.legal.bail_amount` — 50000.0; auto-posted if player has liquidity
- `config.legal.prison_delegation_efficiency` — 0.60; delegate performance during imprisonment
- `config.legal.parole_eligibility_fraction` — 0.50; eligible after 50% of sentence
- `config.legal.double_jeopardy_cooldown_ticks` — 1825; ~5 years before re-filing
- `config.legal.arrest_exposure_hit` — 0.15; immediate exposure increase on arrest
- `current_tick` — for stage duration checks and sentence countdown

## Outputs (to DeltaBuffer)
- `legal_case_deltas[]` — LegalCase state transitions: stage changes, sentence assignment, appeal outcomes
- `player_delta` — PlayerCharacter state changes: exposure increase on arrest, state = imprisoned on conviction, cash -= fine_amount on fined outcome, state = normal on parole
- `npc_deltas[]` — NPCDelta for NPC defendants (identical legal mechanics)
- `evidence_deltas[]` — EvidenceToken(type: public_arrest_record) on arrest
- `consequence_deltas[]` — ConsequenceEntry(legal_process_advances) at each stage transition
- `knowledge_deltas[]` — KnowledgeEntry additions on discovery phase (player learns evidence_token_ids)

## Preconditions
- investigator_engine has completed (InvestigatorMeter statuses are current; investigation progress is finalized for this tick).
- Evidence pool contains all tokens generated by preceding modules this tick.
- LegalCase.evidence_weight reflects current admissible evidence.

## Postconditions
- Every active LegalCase has been evaluated for stage transition this tick.
- Stage transitions respect threshold requirements and timing constraints.
- Convicted defendants have sentence_ticks assigned based on charge_severity.
- Imprisoned defendants have sentence_remaining decremented by 1.
- Acquitted cases are closed; evidence tokens marked inadmissible for re-filing during double_jeopardy_cooldown.
- Player exposure updated on arrest.
- Defence quality reflects player's current legal contacts and resources.

## Invariants
- LegalCaseStage values: investigation=0, arrested=1, charged=2, trial=3, convicted=4, imprisoned=5, acquitted=6, appealing=7, paroled=8, pardoned=9.
- evidence_weight in [0.0, 1.0].
- defence_quality in [0.0, 1.0].
- charge_severity in [1, 5].
- sentence_ticks = charge_severity * base_sentence_ticks_per_severity.
- conviction_probability = evidence_weight * (1 - defence_quality * defence_evidence_discount) * judge_bias_modifier * witness_reliability_modifier, clamped to [0.0, 1.0].
- Player and NPC defendants use identical legal process mechanics; player is not a special case.
- Same seed + same inputs = identical legal process output (deterministic).

## Failure Modes
- Prosecutor NPC not found (dead, fled): case stalls; no stage advancement until new prosecutor assigned. Log warning.
- Judge NPC not found at charged stage: assign from province NPC pool; if none available, case delayed. Log warning.
- Defendant dead during case: case closed as moot. All evidence tokens retained.
- Evidence token referenced by case no longer in pool: evidence_weight recomputed without it; may trigger dismissal.

## Performance Contract
- Sequential execution (not province-parallel); national judiciary.
- Target: < 5ms for full legal processing pass at typical case count (~50-200 active cases).
- O(N_cases) per tick.

## Dependencies
- runs_after: ["investigator_engine"]
- runs_before: [] (Tier 9 terminal)

## Test Scenarios
- `test_arrest_on_evidence_threshold`: Set evidence_weight to 0.36 (above 0.35 threshold) with InvestigatorMeter.status == critical. Verify stage transitions from investigation to arrested and public_arrest_record EvidenceToken generated.
- `test_charges_filed_after_investigation_period`: Create arrested case. Advance by investigation_to_charge_ticks with evidence_weight above charge threshold. Verify stage transitions to charged.
- `test_case_dismissed_on_low_evidence`: Create charged case with evidence_weight 0.20 (below 0.25 dismissal threshold). Verify stage transitions to acquitted.
- `test_trial_conviction_probability`: Create trial case with evidence_weight 0.80, defence_quality 0.50. Verify base_conviction_probability = 0.80 * (1 - 0.50 * 0.40) = 0.64.
- `test_custodial_sentence_at_severity_3`: Convict defendant with charge_severity 3. Verify sentence_ticks = 3 * 365 = 1095 and state = imprisoned.
- `test_fine_at_severity_2`: Convict defendant with charge_severity 2. Verify fine issued (no imprisonment) since severity < custodial_sentence_severity_floor.
- `test_parole_eligibility`: Set imprisoned case with sentence_remaining <= sentence_ticks * 0.50. Verify parole evaluation runs.
- `test_double_jeopardy_prevents_refiling`: Acquit a case. Attempt to re-file within 1825 ticks. Verify case is rejected.
