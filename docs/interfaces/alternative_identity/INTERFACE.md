# Module: alternative_identity

## Purpose
Manages the lifecycle of alternative identities each tick: applies documentation quality decay, processes evidence attribution routing (actions under active alt identity attribute evidence to the alias, not the real actor), handles identity burn mechanics, evaluates identity link discovery by investigators, and tracks witness lists as latent liabilities. The architecture is symmetric — the same AlternativeIdentity struct serves both the player using an alt to evade prosecution and an undercover NPC infiltrating a criminal org.

Not province-parallel: identity records are actor-level and identity link discovery involves cross-province investigative actions (witnesses may be in different provinces from the investigating LE NPC).

## Inputs (from WorldState)
- `player.alt_identities` — all AlternativeIdentity records: id, real_actor_id, alias_name, province_id, status (active/burned/compromised), created_tick, burned_tick, documentation_quality, attributed_evidence_token_ids, witness_npc_ids
- `player.active_alt_identity_id` — 0 = acting as real identity; else = id of active alt
- `significant_npcs` — NPC records for witness NPCs (in witness_npc_ids), LE NPCs performing identity link discovery, undercover NPCs with their own alt identities
- `evidence_pool` — evidence tokens attributed to alt identities (subject_npc_id == alt_identity_id); tokens requiring identity_linked status update on burn
- `informant_records` — InformantRecord for witness NPCs under arrest pressure (can trigger identity link disclosure)
- `current_tick` — for decay timing, burn timing, and created_tick tracking
- `config.identity.documentation_decay_rate` — default 0.001; per-tick decay of documentation_quality without maintenance
- `config.identity.build_identity_depth_rate` — default 0.005; per-action quality gain from build_identity_depth player action
- `config.identity.link_discovery_witness_confidence` — default 0.70; confidence of identity_link KnowledgeEntry from witness disclosure
- `config.identity.link_discovery_forensic_confidence` — default 0.55; confidence from forensic document analysis

## Outputs (to DeltaBuffer)
- `AlternativeIdentityDelta.documentation_quality` — decayed by documentation_decay_rate per tick when not maintained; increased by build_identity_depth_rate on player build action
- `AlternativeIdentityDelta.status` — transitions: active -> burned (on player burn command), active -> compromised (when LE NPC identity_link confidence > 0.5), compromised -> burned (auto-burn or player burn)
- `AlternativeIdentityDelta.burned_tick` — set to current_tick on burn
- `AlternativeIdentityDelta.witness_npc_ids` — appended whenever alt identity interacts with a named NPC (direct observation)
- `EvidenceTokenDelta.subject_npc_id` — for new tokens generated by actions under active alt: set to alt_identity_id instead of real_actor_id
- `EvidenceTokenDelta.identity_linked` — set to false for all attributed tokens on burn; becomes actionable against real actor only on identity_link discovery
- `KnowledgeEntryDelta` — new identity_link KnowledgeEntry on discovery: { type: identity_link, subject_id: alt_id, secondary_subject_id: real_actor_id, confidence: 0.70 (witness) or 0.55 (forensic) }
- `EvidenceTokenDelta` (on link discovery) — all attributed_evidence_token_ids become actionable against real_actor_id immediately

## Preconditions
- Investigator engine has completed; InvestigatorMeter statuses and evidence token attribution are current.
- Informant system has completed; witness NPCs under arrest pressure may have disclosed identity link knowledge.
- Evidence pool contains all tokens generated by preceding modules this tick.

## Postconditions
- Every AlternativeIdentity has documentation_quality decayed by documentation_decay_rate unless maintenance action was performed this tick.
- All evidence tokens generated by actions under an active alt identity have subject_npc_id set to the alt identity id, not the real actor id.
- Burned identities have all attributed evidence tokens set to identity_linked = false. Tokens retain full actionability against the burned identity. Real actor is shielded unless identity_link discovery occurs.
- Compromised identities (LE NPC has identity_link confidence > 0.5) have been flagged; burn-immediately warning issued to player.
- Identity link discovery via witness disclosure, forensic document analysis, or deliberate rival disclosure correctly propagates evidence from alt identity to real actor.
- Only one alt identity can be active at a time per actor.

## Invariants
- IdentityStatus enum: active=0, burned=1, compromised=2.
- Documentation quality in [0.0, 1.0]. 0.0 = street alias only (detected immediately by formal investigation). 0.5 = forged documents (passes routine checks, fails forensic). 1.0 = legend-quality (survives most investigative scrutiny).
- Documentation decay: 0.001 per tick without maintenance. Build rate: 0.005 per player action.
- Evidence attribution rule: IF actor has active AlternativeIdentity THEN token.subject_npc_id = active_alt_identity.id ELSE token.subject_npc_id = real_actor_id.
- Only one active identity at a time. Switching requires physical context (cannot switch mid-scene-card). NPC witnesses to the switch itself know both identities.
- Burn mechanic: immediate on player command. Sets status = burned, burned_tick = current_tick. All attributed tokens set identity_linked = false. Surviving witnesses are permanent latent liability even after burn.
- Identity link discovery triggers: (1) witness NPC arrested/pressured and reveals identity information, (2) forensic document analysis on formal_inquiry against alt identity, (3) rival actor provides link deliberately.
- On identity_link_discovery: KnowledgeEntry { type: identity_link, subject_id: alt_id, secondary_subject_id: real_actor_id, confidence: 0.70 (witness) or 0.55 (forensic) } added to LE NPC KnowledgeMap. All attributed evidence becomes actionable against real_actor_id immediately.
- Witness list is append-only during identity lifetime. Every named NPC interaction under the alt identity adds to witness_npc_ids. Every burned identity's witness list is a permanent latent liability.
- Same seed + same inputs = identical identity output (deterministic).
- All random draws (forensic analysis success, discovery probability) through `DeterministicRNG`.

## Failure Modes
- Alt identity with documentation_quality 0.0 used in formal investigation context: identity detected immediately. Status transitions to compromised. Log diagnostic.
- Witness NPC dead: removed from active liability but historical knowledge entries in other NPCs persist. Log diagnostic.
- Burn command on already-burned identity: no-op. Log warning.
- No LE NPC available for forensic document analysis: identity link discovery via forensic path cannot proceed. Witness path still available. Log diagnostic.
- NaN or negative documentation_quality from edge case: clamp to 0.0, log diagnostic.

## Performance Contract
- Sequential execution (not province-parallel); identity records are actor-level.
- Target: < 3ms total for all identity processing (~5-20 alt identities typical across all actors).
- Must not exceed tick budget share that would push total tick above 200ms target.

## Dependencies
- runs_after: ["investigator_engine", "informant_system"]
- runs_before: ["legal_process"]

## Test Scenarios
- `test_documentation_decays_per_tick`: Alt identity with documentation_quality 0.80, no maintenance. After 100 ticks, verify quality = 0.80 - (100 * 0.001) = 0.70.
- `test_build_identity_depth_increases_quality`: Alt identity with quality 0.50. Player performs build_identity_depth action. Verify quality increases by 0.005 to 0.505.
- `test_evidence_attributed_to_alt_identity`: Player with active alt identity performs action generating evidence. Verify EvidenceToken.subject_npc_id == active_alt_identity.id, not player.id.
- `test_burn_sets_tokens_identity_unlinked`: Player burns alt identity with 5 attributed evidence tokens. Verify all 5 tokens have identity_linked = false. Verify tokens retain actionability against the burned identity.
- `test_identity_link_discovery_from_witness`: Witness NPC in witness_npc_ids is arrested and pressured. NPC discloses identity link. Verify KnowledgeEntry { type: identity_link, confidence: 0.70 } added to arresting LE NPC. Verify all attributed evidence becomes actionable against real_actor_id.
- `test_identity_link_discovery_from_forensic`: LE NPC at formal_inquiry status against alt identity. Forensic document analysis succeeds. Verify KnowledgeEntry { type: identity_link, confidence: 0.55 } added. All attributed evidence becomes actionable against real actor.
- `test_compromised_status_on_high_confidence`: LE NPC gains identity_link KnowledgeEntry with confidence 0.6 (above 0.5 threshold). Verify alt identity status transitions to compromised.
- `test_only_one_active_identity`: Player attempts to activate second alt identity while first is active. Verify first identity is deactivated before second activates.
- `test_witness_list_grows_on_interaction`: Alt identity interacts with 3 named NPCs. Verify witness_npc_ids contains all 3 NPC ids.
- `test_burned_identity_witness_list_persists`: Burn alt identity. Verify witness_npc_ids list is retained and witnesses remain latent liability.
- `test_zero_documentation_detected_immediately`: Alt identity with documentation_quality 0.0 enters formal investigation context. Verify immediate detection and status = compromised.
- `test_determinism_across_runs`: Run 100 ticks with multiple alt identities using fixed seed. Verify bit-identical documentation decay, evidence attribution, and discovery outcomes.
